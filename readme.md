### Постановка задачи
 > Техническое обслуживание станков. Компания занимается ремонтом станков и другого оборудования. Клиентами компании являются промышленные предприятия. Ремонтные работы организованы следующим образом: все станки классифицированы по типам, странам-производителям, годам выпуска и маркам. Все виды ремонта отличаются названием, продолжительностью в днях, стоимостью. Исходя из этих данных, по каждому факту ремонта фиксируется вид станка, дата начала и дата окончания ремонта. Анализ показал, что нужно непросто подразделять станки по типам, а иметь информацию о том, сколько раз ремонтировался тот или иной станок.
### Что находится в данном репозитории?
1. Диаграммы (папка `Diagrams`), в которых описывается работа и структура приложения.
2. Реализация, на основе поставленной задачи и диаграмм (все остальные файлы), на языке C++ с использованием фреймфорка QT.
### Как использовать это приложение?
1. Переименуйте файл `config_example.cpp` -> `config.cpp` и укажите свои данные
2. Приложение готово к использованию, можете собрать его.
3. Добавьте пользователей в базе данных или используйте тестовую заготовку загрузив данные из `models/base_dump` (Запустите приложение и через контекстное меню выберите `Данные`->`Загрузить`->Выберите нужный файл)
